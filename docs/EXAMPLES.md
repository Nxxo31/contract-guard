# Examples — contract-guard v1.0.0

## Compare two OpenAPI specs

```bash
node dist/cli.js compare fixtures/old-api.json fixtures/new-api.json
```

## CLI Options

```bash
# Write report to file instead of stdout
node dist/cli.js compare old.json new.json -o report.md

# Strict mode: exit 1 if breaking changes exist (for CI gates)
node dist/cli.js compare old.json new.json --strict
echo $?  # 0 if no breaking, 1 if breaking changes found

# Hide SAFE CHANGES section (focus on actionable changes)
node dist/cli.js compare old.json new.json --no-safe

# Combine flags
node dist/cli.js compare old.json new.json --strict --no-safe -o report.md
```

The repository includes two real OpenAPI 3.0 specs in `fixtures/`:

| File | Description |
|------|-------------|

| `fixtures/new-api.json` | Modified spec: breaking + warning + safe changes |

**Changes in the fixture comparison:**

## GitHub Actions — strict mode (CI gate)

This pattern fails the build when breaking changes are detected, preventing accidental merges:

```yaml
name: API Contract Check
on:
  push:
    branches: [main]
  pull_request:

jobs:
  contract:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install
        run: npm ci

      - name: Build
        run: npm run build

      - name: Contract check
        run: node dist/cli.js compare specs/old.json specs/new.json --strict
```

## GitHub Actions — generate and upload report

```yaml
- name: Generate contract report
  run: node dist/cli.js compare specs/old.json specs/new.json -o contract-report.md

- name: Upload report artifact
  uses: actions/upload-artifact@v4
  with:
    name: contract-report
    path: contract-report.md
    retention-days: 30
```

## GitHub Actions — post report as PR comment

```yaml
- name: Generate report
  id: report
  run: node dist/cli.js compare specs/old.json specs/new.json -o report.md

- name: Post comment
  uses: actions/github-script@v7
  with:
    script: |
      const fs = require('fs');
      const report = fs.readFileSync('report.md', 'utf-8');
      github.rest.issues.createComment({
        issue_number: context.issue.number,
        owner: context.repo.owner,
        repo: context.repo.repo,
        body: '## Contract Guard Report\n\n' + report
      });
```

## Compare specs across git branches

```bash
# Get old spec from main branch
git show main:openapi.json > /tmp/old-spec.json

# Use current working directory spec as new
cp openapi.json /tmp/new-spec.json

# Run comparison
node dist/cli.js compare /tmp/old-spec.json /tmp/new-spec.json --strict
```

## Compare specs across git tags (for release notes)

```bash
PREV_TAG=$(git describe --tags --abbrev=0 HEAD^)
node dist/cli.js compare \
  <(git show $PREV_TAG:openapi.json) \
  openapi.json \
  --strict \
  -o CHANGELOG.md
```

## Programmatic Usage

```ts
import { loadSpecFromFile, normalizeSpec } from './parser';
import { diffSpecs } from './diff';
import { classifyChanges } from './rules';
import { buildReport } from './report';

const oldSpec = normalizeSpec(loadSpecFromFile('old-api.json'));
const newSpec = normalizeSpec(loadSpecFromFile('new-api.json'));
const diff = diffSpecs(oldSpec, newSpec);
const classified = classifyChanges(diff.changes);

// Full report object
const report = buildReport(diff, classified, {
  includeSafeChanges: true,
  strict: false
});

console.log(report.markdown);          // Markdown string
console.log(report.hasBreakingChanges); // boolean
console.log(report.hasWarnings);        // boolean
console.log(report.summary);            // "N breaking, N warning(s), N safe"

// Exit 1 if breaking changes exist (CI use)
if (report.hasBreakingChanges) {
  process.exit(1);
}
```

## Exit Codes

| Code | Meaning | When it occurs |
|------|---------|----------------|
| `0` | Success | Normal execution; report printed |
| `1` | Breaking changes in `--strict` mode | `--strict` flag used and breaking changes detected |
| `2` | Error | Invalid spec, file not found, JSON parse error |
